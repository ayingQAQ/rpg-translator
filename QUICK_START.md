## 🚀 快速开始

### 方式一：一键汉化（推荐）

适合：整个游戏目录的所有文本文件

**操作步骤：**
1. 启动程序：`python gui_main.py`
2. 点击 "Load Game Directory" 加载游戏文件夹
3. 选择翻译引擎和目标语言
4. 点击绿色的 "🚀 一键汉化" 按钮
5. 等待自动完成（会显示进度条）
6. 完成后直接进游戏测试！

**特点：**
- ✅ 自动扫描所有文本文件
- ✅ 自动备份原文件（.backup后缀）
- ✅ 自动翻译并覆盖
- ✅ 支持子目录递归
- ✅ 多线程并发，速度飞快

---

### 方式二：一键恢复

适合：想要还原原版、切换回原语言、或者汉化出错时

**操作步骤：**
1. 加载包含备份文件的游戏目录
2. 点击红色的 "⏪ 一键恢复" 按钮
3. 确认警告信息
4. 等待自动恢复完成

**特点：**
- ✅ 自动扫描所有 .backup 备份文件
- ✅ 兼容两种备份命名格式（.backup.json / .backup_123456.json）
- ✅ 恢复后保留备份文件（后悔药设计）
- ✅ 一键还原，无需手动查找

---

### 方式三：单文件翻译

适合：只想翻译特定文件或查看翻译结果

**操作步骤：**
1. 启动程序：`python gui_main.py`
2. 点击 "Load Game Directory" 加载游戏文件夹
3. 点击 "Extract Game Text" 提取文本
4. 在文件列表中双击要翻译的文件
5. 点击 "Translate All" 或 "Translate Selected"
6. 查看翻译结果并保存

## 💡 使用技巧

### 如何选择翻译引擎？

- **Google翻译**：免费，支持多语言，但质量一般
- **DeepL**：翻译质量最好，但可能需要API密钥
- **百度翻译**：中文相关翻译较好
- **Local**：使用本地模型，无需联网但速度较慢

### 翻译质量优化

1. **批量处理**：对于大型游戏，优先使用"一键汉化"
2. **延迟设置**：如果遇到API限制，可在 `core/translator.py` 中增加 `delay` 值
3. **备份重要**：首次翻译前手动备份游戏文件，虽然程序会自动备份
4. **分批翻译**：如果游戏文件特别多（几千个），可以分文件夹进行

### 常见问题

**Q: 一键汉化后游戏没变化？**
A: 检查：
   - 是否翻译完成（查看日志）
   - 游戏是否使用翻译后的文件
   - 文件编码是否正确

**Q: 翻译速度还是慢？**
A: 尝试：
   - 检查网络连接
   - 换个翻译API试试
   - 减少max_workers值（避免被限流）

**Q: 翻译出错怎么办？**
A: 查看：
   - 错误日志（程序目录下的日志文件）
   - 文件格式是否被支持
   - 文件编码是否为UTF-8

## 🔧 高级配置

### 调整线程数

编辑 `core/translator.py`：
```python
max_workers = 10  # 免费API建议5-10，付费API可20-50
```

### 调整延迟

编辑 `gui_main.py` 的 OneClickTranslateThread：
```python
delay = 0.1  # 多线程下可更低，单线程需要0.5以上
```

### 添加文件格式支持

编辑 `parsers/__init__.py`，添加新的解析器
